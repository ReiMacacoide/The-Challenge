About Saxon-JS

Saxon-JS is an XSLT 3.0 run-time written in pure JavaScript. It's designed to execute stylesheets compiled by Saxon-EE into an intermediate form (called the Stylesheet Export File or SEF), as shown in Figure 1.

The Saxon product effectively contains a compile time and a run time component. The compiler runs on the server, and generates the Stylesheet Export File (SEF), which is an intermediate representation of the compiled and optimized stylesheet in a custom XML format. It's the same compiler whether you want to execute the transformation in the browser or on the server. Saxon-JS, running on the browser, reads in the SEF and executes it.

Because people want to write rich interactive client-side applications, Saxon-JS does far more than simply converting XML to HTML, in the way that the original client-side XSLT 1.0 engines did. Instead, the stylesheet can contain rules that respond to user input, such as clicking on buttons, filling in form fields, or hovering the mouse. These events trigger template rules in the stylesheet which can be used to read additional data and modify the content of the HTML page.

We're talking here primarily about running Saxon-JS in the browser. However, it's also capable of running in server-side JavaScript environments such as Node.js (not yet fully supported in this release).

Because Saxon-JS only handles the run time aspects of XSLT transformation, without all the code needed for static analysis and compilation, the product is much smaller and faster than it would otherwise be. There's no point doing all this work every time someone loads a web page, when it can be done once by the developer. In addition, distributing your code in the form of a stylesheet export file gives you some intellectual property protection: more than it would get if you distributed it in JavaScript or in source XSLT.

Saxon-JS has almost-complete support for XSLT 2.0 and XPath 2.0, and also implements a great deal of XSLT 3.0, XPath 3.0, and XPath 3.1, including support for maps, arrays, and JSON.
