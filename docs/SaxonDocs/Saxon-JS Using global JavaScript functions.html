Using global JavaScript functions

Global JavaScript functions (those belonging to the global Window object) can be called directly. For example if a function has been declared as <script type="text/javascript" language="javascript"> function square(x) { return String(x*x) } </script> then it can be called from within an XPath expression as js:square('5') where js is bound to the namespace http://saxonica.com/ns/globalJS.

Saxon-EE converts the function call js:square('5') at compile time into the function call ixsl:call(ixsl:window(), 'square', ['5']). This means that there is no compile-time check that a function with the name "square" actually exists. The test function-available('js:square') returns true whether the function exists or not: in a sense, the XPath function js:square always exists, but it returns a dynamic error if the underlying JavaScript function does not exist.

Functions nested in JavaScript objects can be obtained by using dot separator characters in the function name: for example, js:Math.random() calls the function to generate a random number.

See Type Conversion for details on how the supplied parameters are converted from XDM to JavaScript, and how the returned JavaScript objects are converted to XDM.
