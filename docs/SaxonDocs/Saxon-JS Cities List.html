Cities List

To try out the cities list application, visit here.

The first page contains an alphabetical index. Click on a letter to see the list of US cities beginning with that letter. Try clicking on column headings to sort the table by a chosen column.

This example demonstrates the use of various new features of XPath 3.1, including reading JSON resources, and using maps and arrays. To see how it works, view the source of the HTML page ("view source" in your browser), and examine the XSLT stylesheet. Particular features to note:

  • The HTML page contains two script elements, one to identify the Saxon-JS JavaScript library, the other to invoke a transformation using the specified compiled stylesheet and source document.

  • The XSLT code addresses an area of the HTML page to be updated using xsl:result-document. The href attribute is used to specify the ID of the target HTML element where the generated fragment of HTML is to be inserted.

  • There is one event handler in the stylesheet - the template rule with mode="ixsl:onclick". This template rule is used to sort the contents of the table by column heading. User-defined attributes such as data-order and data-type are used to retain state information in the HTML tree, so that the sorting process can decide whether to sort ascending or descending, and whether to sort alphabetically or numerically.

The source cities datasets are stored in JSON files (for example, A.json). The original sample datasets were obtained from http://openweathermap.org. There was some server side preparation of these source files (using XSLT) - firstly to extract cities in the US, and then to split into separate files by initial letter of the city name. Together these JSON files contain nearly 2MB of data. Each file consists of a JSON array of objects containing the cities data. These are read using the XPath 3.1 function json-doc() to return the data in an XPath array of XPath maps.
