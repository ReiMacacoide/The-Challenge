ixsl:set‑property

Sets the value of properties for JavaScript objects (including DOM nodes).

Requires Saxon-EE. Implemented since Saxon 9.7.0.7.

Category: instruction
Content: none
Permitted parent elements: any XSLT element whose content model is sequence-constructor; any literal result element

Attributes

name

eqname

Either a property name or a dot (character '.') separated list of names.

select

expression

Provides the new property value.

object?

expression

The object the property belongs to — when no object attribute is present, the Window object is used.

Details

An error will be raised at run-time if the specified object does not exist. If the specified property does not exist, then there is no error, but a warning is output in the console log.

Examples

The following example shows a function that can be used to set the hash property of the Window.location object (the part of the URL after a '#' symbol). The new property value is the argument to the function prefixed with '!'.

<xsl:function name="f:set-hash"> <xsl:param name="hash"/> <ixsl:set-property name="location.hash" select="concat('!',$hash)"/> </xsl:function>
