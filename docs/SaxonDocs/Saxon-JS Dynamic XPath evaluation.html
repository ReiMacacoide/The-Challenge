Dynamic XPath evaluation

From Saxon-JS 0.9.1 dynamic evaluation of XPath expressions within an XSLT transform is available through runtime support for the xsl:evaluate expression. The code to support this will be dynamically loaded from the location lib/xpath/XPathJS.min.js relative to the Saxon-JS script location. This load will only occur once, on the first invocation of an xsl:evaluate instruction. (Thus no overhead is involved if you don't use dynamic evaluation.)

The evaluation assumes XPath 3.1 syntax and semantics.

It is not intended that the implementation will handle these features:

  • Any schema-awareness
  • XPath 1.0 backwards compatibility
  • xsl:fallback child

The XPath expression should not contain:

  • Embedded comments
  • Any higher order functions including function name references and inlined function bodies
  • Any of the other general restrictions on Saxon-JS, such as unsupported functions (e.g. fn:sort())
