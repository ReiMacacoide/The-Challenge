ixsl:source

Returns the global context item - the document node supplied as input to the transformation using a parameter such as sourceNode, sourceLocation, or sourceText, if such a source document was supplied, or the empty sequence otherwise.

Requires Saxon-EE. Implemented since Saxon 9.7.0.7.

source() ➔ node()?

There are no arguments

Result

node()?

Namespace

http://saxonica.com/ns/interactiveXSLT

Notes on the Saxon implementation

Available since Saxon 9.7.0.7.

Details

Useful for retrieving the source document from within an event handling template, as the context item will be the object whose event is being handled - i.e. an element of the HTML page for a user input event, or an object outside the HTML DOM for a client system event.

If there is no global context item, then the function returns the empty sequence.

Examples

A user click event on the "go" button initiates apply-templates processing of the source document to produce content for the "target" div in the HTML page:

<xsl:template match="button[@id='go']" mode="ixsl:onclick"> <xsl:result-document href="#target" method="ixsl:replace-content"> <xsl:apply-templates select="ixsl:source()"/> </xsl:result-document> </xsl:template>
