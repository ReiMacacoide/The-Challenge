Comparison with Saxon-CE

Saxon-JS can be seen as a rewrite of Saxon-CE - the XSLT 2.0 processor for use in the browser. While Saxon-JS is written in pure JavaScript, Saxon-CE was produced by cross-compiling a cut-down version of the Java product to JavaScript (using Google's GWT cross-compiler). Saxon-JS is smaller and faster, it implements many useful XPath 3.0 features including support for JSON, it can run in non-browser environments such as Node.js (not yet fully supported in this release), and it's based on the latest Saxon code base.

Saxon-CE pioneered the idea of using XSLT for handling interactive user events, which is replicated in almost identical form in Saxon-JS. Because Saxon-JS implements the XSLT 3.0 features for handling maps, arrays, and JSON, the mechanisms used for calling out from XSLT to JavaScript and for accessing properties of the HTML DOM have been redesigned to take advantage of these constructs. See the Development and Saxon Interactive Extensions sections for details.

For information on updating applications which were originally built to use Saxon-CE, see Migration from Saxon-CE.
