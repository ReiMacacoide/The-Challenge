ixsl:apply

Applies a JavaScript function.

Requires Saxon-EE. Implemented since Saxon 9.7.0.12.

apply($function as item(), $arguments as array(*)) ➔ item()?

Arguments

 

$function

item()

JavaScript function object

 

$arguments

array(*)

Arguments to the function, supplied as an array

Result

item()?

Namespace

http://saxonica.com/ns/interactiveXSLT

Notes on the Saxon implementation

Available since Saxon 9.7.0.12. Requires Saxon-JS 0.9.1 or later.

Details

The first argument is a JavaScript function object (for example one returned from ixsl:eval()). The second argument is an XPath array supplying the actual arguments to the function (if no arguments are to be supplied to the function, then this second argument should be an empty array). The result of calling the function is returned as the result of ixsl:apply().

See Type Conversion for details on how the supplied parameters in $arguments are converted from XDM to JavaScript, and how any returned JavaScript objects are converted to XDM.

An error will be raised at run-time if the object supplied as the first argument is not a JavaScript function object, or if the function cannot be called with the supplied arguments.

Examples

Example using ixsl:apply() with ixsl:eval(). When evaluated, the following returns the boolean value true:

<xsl:variable name="is-string-fn" select="ixsl:eval('(function isString (item) { return typeof item == ''string''})')"/> <xsl:sequence select="ixsl:apply($is-string-fn, ['a string'])"/>
