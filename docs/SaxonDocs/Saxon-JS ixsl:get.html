ixsl:get

Gets a property of a JavaScript object (which may be an object returned by another extension function, or a node in the HTML DOM).

Requires Saxon-EE. Implemented since Saxon 9.7.0.7.

get($object as item()?, $property as xs:string) ➔ item()?

Arguments

 

$object

item()?

JavaScript object

 

$property

xs:string

Either a property name or a dot (character '.') separated list of names

Result

item()?

Namespace

http://saxonica.com/ns/interactiveXSLT

Notes on the Saxon implementation

Available since Saxon 9.7.0.7.

Details

The first argument is the object on which the property is defined. The second argument is the property name; or to obtain a nested property, use dot separator characters within this second argument (see Properties of JavaScript objects for more information). The result of calling the function is returned as the result of ixsl:get().

If the specified object or property does not exist, then the function returns the empty sequence (with a console log warning in the latter case). (Note the difference to the similar function ixsl:call(), for which a run-time error is raised in this case.)

See Type Conversion for details on how the returned property (itself a JavaScript object) is converted to XDM.

In some cases, $object?prop (shorthand for map:get($object, "prop")) is interchangeable with ixsl:get($object, "prop"). The requirement for this is that $object is a JavaScript object which is not a more specific type which is recognised and converted to an XDM "equivalent" (e.g. null, Node, Array), and so within an XPath expression it is treated as a special wrapped (JSValue) object, which behaves something like a "read-only" XPath map. See Properties of JavaScript objects and JavaScript to XDM Conversion for details.

Examples

Example 1

The offsetHeight property of an HTML element $node is obtained using:

ixsl:get($node, "offsetHeight")

Example 2

Example using dot separator characters within $property to obtain a nested property. The hash property of the location property of the Window object is obtained using:

ixsl:get(ixsl:window(), "location.hash")

For improved readability, consider using the XPath 3.1 arrow operator:

ixsl:window() => ixsl:get("location.hash")

Alternatively, the XPath 3.1 lookup operator can be used here:

ixsl:window()?location?hash

Example 3

As a more complicated example, suppose we wish to get the X coordinate of the first touch point of a touch event. In JavaScript, this would be accessed as event.touches[0].clientX, where event is the TouchEvent object. In an IXSL event handling template, the property can be obtained using either of the following:

ixsl:get(ixsl:event(), 'touches.0.clientX') ixsl:event()?touches?0?clientX

See Properties of JavaScript objects for more details.
