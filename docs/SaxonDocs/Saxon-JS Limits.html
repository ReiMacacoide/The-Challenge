Limits

Tail calls are not optimized, so functions that process a sequence using head-tail recursion will only be able to process sequences up to a limit of a couple of hundred items.

xs:float is implemented using the JavaScript Number data type (same as xs:double). Saxon-JS attempts to reduce the precision of results to the appropriate level, but does not always give the precise result required by the IEEE floating point rules.

xs:integer and xs:decimal: Small integers are represented internally using JavaScript Number objects (double precision floating point). Larger integers, and all decimals, are represented using the Big.js library. This means there is effectively no limit to the size of integers and decimals that can be held.

xs:dateTime, xs:date, xs:time, etc, are implemented using the JavaScript Date library. They therefore inherit the limitations of this library, e.g. in the range of years and the precision of fractional seconds supported. Saxon-JS maintains an xs:dateTime as a combination of a Date object and a timezone offset. The values of current-dateTime() and similar functions are obtained using Date.now(), and the implicit timezone is the timezone offset in the result of Date.now(), which generally reflects the timezone of the system clock on the user's machine.
