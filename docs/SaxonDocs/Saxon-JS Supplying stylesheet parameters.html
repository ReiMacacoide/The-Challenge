Supplying stylesheet parameters

Stylesheet parameters are supplied using the stylesheetParams option on the call of SaxonJS.transform. The values that can be supplied depend on the declared type of the parameter, as follows:

Declared XDM type Supplied JavaScript type Conversion
xs:string or xs:untypedAtomic Any value

Applies the toString() method.
xs:boolean Boolean

No conversion.
xs:double Number

No conversion.
node() DOM Node (XML or HTML)

No conversion.
map(*) Object

The properties of the JavaScript object become key-value pairs within the map, where the key is the property name (as a string), and the value is obtained by applying the JavaScript to XDM conversion rules (as described at JS/XDM Type Conversion) recursively, using "Case 2" conversion for nested objects.
array(*) Array nested in an Array

JavaScript to XDM conversion using the rules given at JS/XDM Type Conversion, using "Case 2" conversion for nested objects.
X?, X+, X*, where X is one of the above Array (either an empty Array or Null can be used for an empty sequence)

The items of the supplied array are converted to type X using the above rules (so any nested arrays become XPath array items).

If no type is declared, or if the type is declared as item(), then Saxon-JS converts the supplied value as described in JS/XDM Type Conversion (using "Case 1" conversion for 'other' JavaScript objects).

If the declared type is anything other than the above, then Saxon-JS will raise an error.
