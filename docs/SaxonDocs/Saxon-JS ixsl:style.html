ixsl:style

Returns an XPath map containing the style properties of an HTML element node.

Requires Saxon-EE. Implemented since Saxon 9.7.0.7.

style($node as item()?) ➔ map(*)

Arguments

 

$node

item()?

The HTML element node

Result

map(*)

Namespace

http://saxonica.com/ns/interactiveXSLT

Notes on the Saxon implementation

Available since Saxon 9.7.0.7.

Details

Used to access the properties of the JavaScript computed style object (CSSStyleDeclaration) associated with an element node in the HTML DOM tree. This object is obtained using window.getComputedStyle() and includes all CSS declarations (such as style rules in the head section, or external style sheets), rather than just those from the element's inline style attribute (obtained using HTMLElement.style).

The style information is returned as an XPath map of type map(xs:string, xs:string?). The entries in the map are key-value pairs, in which the keys are strings corresponding to the style property names, in kebab-case (e.g. 'font-size'). The corresponding values are strings containing the style property value (or the empty sequence if the style property value is undefined or null). The value of an entry in a map can be obtained using the XPath "?" operator, e.g. to obtain the value of the 'hidden' style property of the current node use ixsl:style(.)?hidden.

If the specified node does not exist, or is not an element node, then the function returns an empty map.

Examples

The example below demonstrates checking computed style properties of an HTML element, and setting the corresponding properties under certain conditions. Note the mismatch between getting style properties using kebab-case names, and setting them using camel-case names:

<xsl:template match="div[@id='list']//li" mode="ixsl:onmouseover"> <xsl:if test="ixsl:style(.)?font-weight = ('normal', '400')"> <ixsl:set-style name="fontWeight" select="'bold'"/> </xsl:if> <xsl:if test="ixsl:style(.)?font-size = '12px'"> <ixsl:set-style name="fontSize" select="'20px'"/> </xsl:if> </xsl:template>
